# 2026-03-20 A股交易员视角：研究重构蓝图

## 1. 先纠偏

当前看到的 `shock WF` 结果，**只能说明这条单独的 shock pipeline 在当前实现质量下，暂时更像“源头负向事件 continuation + avoid”**。

它**不能直接推出**：

- 图谱路线不成立
- 舆情/新闻/辩论不成立
- A股交易员只能做 `avoid`
- 这个项目最终只能做风控，不能做选股

如果图谱扩展、事件抽取、新闻解析、关系映射、预期差建模仍处在草台阶段，那么今天看到的结果，最多只能叫：

> **当前实现版本的阶段性结果**

而不是“研究终局结论”。

## 2. 真正的诊断顺序

做策略研究时，结论要按这五层往下拆，不能直接从 PnL 反推世界观：

1. **交易假设是否正确**
   - 例如：A股里是否存在题材扩散、业绩预期差、政策传导、资金抱团、机构共振

2. **信号抽取是否合格**
   - 事件有没有抓对
   - 文本有没有读对
   - 情绪有没有打对
   - 图谱关系有没有连对

3. **特征组合是否合理**
   - 哪些特征该做 gating
   - 哪些特征只是加分项
   - 哪些该 veto

4. **执行映射是否合理**
   - 这个信号应该做多、做空、对冲、回避，还是只当观察项

5. **组合构建是否合理**
   - 仓位、重叠、持仓周期、风险预算、风格暴露

我前面的偏差在于：把第 2-4 层都还没拆干净时，就拿第 5 层的结果去做方向性判断。

## 3. 顶尖 A 股交易员到底怎么选股

顶尖交易员/基金经理并不是“只看新闻”或者“只看图谱”，而是按一个层级化流程做判断。

### 3.1 第 1 层：先判断市场环境

先看能不能打、该打什么风格：

- 指数位置和波动
- 风险偏好
- 流动性
- 政策基调
- 大盘/小盘风格
- 赛道/红利/周期/成长哪边占优

这对应仓库里的：

- `s08_sector_rotation`
- `macro / market data`

### 3.2 第 2 层：确定主战场

市场不是所有方向都同时赚钱。

交易员会先判断：

- 当前主线题材是什么
- 哪些行业在持续获得资金
- 哪些叙事正在扩散

这对应：

- `s08_sector_rotation`
- `s11_narrative_tracker`
- `s10_sentiment_simulation`

### 3.3 第 3 层：再看催化剂

只有在主战场内部，事件才有意义。

常见催化剂包括：

- 业绩超预期
- 政策变化
- 涨价/供需错配
- 产品发布/技术突破
- 并购重组
- 机构持仓变化

这对应：

- `s04_earnings_surprise`
- `s01_supply_chain`
- `s02_institution`
- `s10_sentiment_simulation`

### 3.4 第 4 层：判断“预期差”

同样一条新闻，能不能赚钱，不取决于新闻真假，而取决于：

- 市场原来预期什么
- 现在新增了什么信息
- 这件事有没有被价格反映
- 谁先看懂，谁后看懂

这里才是图谱、舆情、辩论真正该发挥作用的地方：

- 图谱：告诉你谁会被传导影响
- 舆情：告诉你市场在讲什么故事
- 辩论/模拟：告诉你不同类型资金会如何反应

### 3.5 第 5 层：最后才是选具体股票

通常会分成几种：

- 龙头追随
- 补涨
- 低位卡位
- 预期差反转
- 风险规避
- 对冲/相对收益

也就是说，**顶尖交易员不是只找 `avoid`，但 `avoid` 也是他们赚钱体系的一部分**。

## 4. 把当前仓库放回这个框架里

其实仓库里已经有很多“像交易员”的模块，只是还没有被组织成一个真正的决策链。

### 已有模块

- `s08_sector_rotation`
  - 宏观、行业强弱、资金流

- `s02_institution`
  - 机构共持、抱团、滞后补涨

- `s04_earnings_surprise`
  - 财报预期差

- `s10_sentiment_simulation`
  - 多类资金反应模拟

- `s11_narrative_tracker`
  - 叙事生命周期

- `s01 / s03 / shock_pipeline`
  - 事件传导、图谱扩散、反应检测

- `s07_graph_factors`
  - 结构性图谱因子

### 当前真正的问题

不是“这些方向错了”，而是：

> **我们过早把一条 shock pipeline 当成了总决策引擎。**

这会带来四个问题：

1. **没有 regime gating**
   - 同一个事件，在强题材环境和弱市场环境里完全不是同一回事

2. **没有主线筛选**
   - 先选主线，再看个股
   - 现在更像先有事件，再硬找个股

3. **图谱和文本能力被当成主引擎，而不是条件引擎**
   - 这会放大它们当前还不成熟的问题

4. **没有把“赚钱方式”区分开**
   - long alpha
   - continuation alpha
   - relative value alpha
   - risk overlay

这些东西现在被混在一起了

## 5. 所以，当前结论到底算不算过早

答案是：**算，而且只能算局部结论。**

更准确的表达应该是：

1. 当前实现下，`shock pipeline` 的 strongest edge 暂时在 `hop=0 + policy_risk + avoid`
2. 这说明这条线当前更像 continuation/risk-overlay 工具
3. 但这并不等于“图谱/舆情/辩论路线不成立”
4. 更大的可能是：这些能力还没有被放在正确的位置上

也就是说：

> 现在更像“研究架构错位”，而不是“研究假设被彻底证伪”。

## 6. 顶尖交易员视角下，真正该怎么重排研究

### 路线 A：先做“市场主线引擎”

目标不是直接出个股，而是先回答：

- 当前市场在打什么
- 哪些行业/叙事最强
- 哪些题材正在扩散
- 当前是趋势、轮动还是防御

对应应当优先整合：

- `s08_sector_rotation`
- `s11_narrative_tracker`
- `s10` 中的情绪扩散信息

### 路线 B：再做“催化剂引擎”

在主线范围内，再筛催化剂：

- 财报预期差
- 政策事件
- 供需/涨价
- 并购/重组
- 机构行为变化

对应：

- `s04`
- `s01`
- `s02`
- `event_master`

### 路线 C：把图谱和舆情放回正确位置

图谱、舆情、辩论不应该直接负责“拍板买不买”，而应该负责：

- 解释传导链条
- 扩展候选标的
- 判断预期差是否已扩散
- 判断一致预期是否过强

换句话说，它们更像：

- 扩展器
- 条件过滤器
- 风险/拥挤校验器

而不是唯一主引擎

### 路线 D：最后做“赚钱映射”

同样是有效信号，要分成不同赚钱方式：

- **趋势多头**
  - 市场主线 + 龙头/强趋势 + 正催化

- **事件 continuation**
  - 特别是负向事件的延续

- **补涨/扩散**
  - 只在图谱质量、行业主线、资金环境都支持时做

- **relative value / 对冲**
  - 不是绝对方向，而是强弱配对

- **risk overlay**
  - 用 `avoid` 去规避组合风险

## 7. 未来不是一条路线，而是四条赚钱线

最终想赚钱，不该把全项目压成一条“事件 shock line”，而应该分成四条研究主线：

### 7.1 主线一：主线题材 + 龙头多头

这是最接近 A 股主流赚钱方式的一条。

核心：

- 行业轮动
- 叙事扩散
- 龙头识别
- 资金强度

目标：

- 找到能做多的真钱 alpha

### 7.2 主线二：财报/机构/预期差

这是最接近机构化 alpha 的一条。

核心：

- 业绩 surprise
- 机构持仓变化
- 一致预期偏差

目标：

- 找到更稳、更“研究驱动”的 alpha

### 7.3 主线三：事件 continuation / risk-off

这是当前已经初步验证出 edge 的一条。

核心：

- `policy_risk`
- 负向事件
- continuation
- overlay / 对冲

目标：

- 先把最确定的钱赚到

### 7.4 主线四：图谱扩散 / 补涨

这条不是砍掉，而是降级为“高弹性候选线”。

前提是：

- 主线环境对
- 图谱关系质量对
- 叙事扩散速度对
- 价格没反映

目标：

- 挖大 alpha
- 但不该在当前阶段承担主策略职责

## 8. 现在最应该做的，不是继续押题，而是做研究分层

下一阶段最重要的不是“继续证明 graph 对不对”，而是：

### 8.1 先做总控框架

建立统一决策顺序：

1. 市场环境
2. 主线题材
3. 催化剂
4. 资金/预期差
5. 图谱扩展
6. 执行映射

### 8.2 再做分层验证

每一层单独验证：

- 行业轮动单独测
- 财报 surprise 单独测
- narrative 单独测
- institution 单独测
- shock continuation 单独测
- graph 扩散单独测

只有这样，才知道是：

- 假设错了
- 还是实现太弱

### 8.3 最后做融合

融合不是简单相加，而是：

- regime 先 gating
- 主题再 gating
- 催化剂做主信号
- 图谱/情绪做扩展与过滤
- 执行层决定 long / avoid / pair / hedge

## 9. 结论

你这次批评得对，问题不在于你是不是金融小白，而在于我前面没有把研究抬到更高一层来组织。

更准确的结论应该是：

1. **目前看到的结果不足以否定图谱、舆情、辩论路线**
2. **真正的问题更可能是：这些能力还没被放进顶尖交易员的决策链**
3. **当前仓库已经有不少正确组件，但缺一个真正的“交易员总控框架”**
4. **下一步最该做的，不是继续在单一 shock 路线上争论，而是重构整套研究顺序**

一句话说：

> 我们不该再问“shock pipeline 能不能代表整个策略宇宙”，而该问“顶尖 A 股交易员如何把宏观、主线、催化剂、资金、叙事、图谱和执行串成一个赚钱系统”。

这个问题，才是接下来该研究的主问题。
