这份文档用于说明 `MiroFish` 当前“底层数据建设”已经接入了哪些数据、数据来自哪里、落到哪里、主要字段是什么、能支持什么研究，以及当前覆盖率和质量状态。

## 6. 当前质量边界

### 已解决的问题

- manifest 路径已改成仓库可迁移路径，不再依赖旧机器绝对路径
- graph 和 sentiment 不再把降级数据伪装成正常数据
- filings 已经切换到官方 `CNInfo` 逐股窗口查询
- market hot topic 已切到 `Baidu + Xueqiu`

### 仍需注意的点

- `skipped_filing_other = 2130` 仍然较高
  - 剩余高频主体大多是独立董事述职、会议决议、H 股公告、ESG/可持续发展报告
  - 这些多数是低交易信号内容，不适合继续无差别塞入事件层
- `event` 还没有做到 `800 / 800`
  - 原因是并非所有股票在窗口内都形成可交易事件
- `filings` 也不会天然达到 `800 / 800`
  - 原因是很多股票在近 `30` 天窗口内没有官方公告

## 7. 建议理解方式

- `price / news / sentiment / graph` 可以看作“底座主干已齐”
- `filings / events` 可以看作“正在向更高覆盖和更高信噪比推进”
- `audit` 是研究入口的最终开关，不建议绕开它直接假定全股票都可研究
