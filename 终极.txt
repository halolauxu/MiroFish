Phase 2. Alpha 能力地图
主线 A: 知识图谱供应链因子
经济假设
供应链网络中处于"关键节点"位置的企业（高入度/出度、居间中心性高）享有持续的定价权和需求稳定性溢价。PageRank衡量的中心性与超额收益正相关，且这一效应在供应链关系重构时期（如国产替代、能源转型）尤为显著。

事件链逻辑


供应链图谱结构 → PageRank中心性计算 → 截面排序
→ 高中心性 = 更强定价权 + 更稳定现金流 → 更高估值/更低波动 → Alpha
所需结构化字段

字段	来源	频率
supply_chain_centrality	图谱PageRank	月度（图谱更新时）
betweenness_centrality	图谱拓扑	月度
upstream_count / downstream_count	图谱度数	月度
roe_ttm, pe_percentile	基本面	季度
momentum_20d, reversal_5d	市价	日度
当前状态: S07已实现，回测A级（T-40窗口80%胜率，+11.68%，Sharpe 1.87）。
主要风险: 图谱仅96条边，中心性计算严重不足；需扩充至≥2000条边才能使PageRank有统计意义。

主线 B: 供应链冲击传播
经济假设
上游企业公告涨价/限产/断供时，下游受影响企业的股价反应存在系统性滞后（1-5个交易日）。这一滞后来自：①信息处理成本；②分析师覆盖不均；③机构调研延迟。LLM+图谱可以比市场更快完成"上游冲击→下游影响路径→受影响程度"的全链路推理。

事件链逻辑


上游公司公告（涨价/限产/扩产） 
→ LLM确认事件类型 + 量化幅度
→ 图谱遍历 SUPPLIES_TO 边找下游
→ LLM评估: 成本转嫁能力 × 替代供应商可得性 × 库存缓冲期
→ 股价是否已反应（过去5日涨幅 vs 预期影响）
→ 未反应 → 买入/回避信号

cost_passthrough_score	LLM 0-1打分	必须
already_reacted_flag	价格动量检测	必须
graph_hop_distance	图谱路径长度	重要
当前状态: 策略存在但0信号——根因是图谱稀疏（96条边，覆盖极少数股票对）。不是LLM问题，是图谱问题。
主要风险: ①公告API只抓到有限公告；②图谱覆盖不足导致遍历找不到下游；③A股公告质量良莠不齐（语义模糊）。

主线 C: 多智能体叙事智能
经济假设
A股散户占比高（~70%换手），投资叙事（"ChatGPT概念"、"低空经济"、"国产替代"）的生命周期可预测：萌芽→扩散→成熟→衰退。Alpha来自比市场共识早1-3周识别叙事阶段转折点。MiroFish的OASIS多智能体仿真可以模拟散户/机构/媒体三类角色对信息的传播响应，输出比单一LLM更接近市场真实扩散动态的预测。

事件链逻辑


新闻/公告文本 + 资金流 + 涨停板数据
→ OASIS多智能体讨论（散户角色×N + 机构角色×M + 媒体角色×K）
→ 仿真情绪扩散曲线
→ 叙事生命周期阶段判断（萌芽/扩散/成熟/衰退）
→ 萌芽期主题龙头 → 买入; 衰退期纯主题股 → 回避

当前状态: S10基础框架存在，S11叙事跟踪存在，但两者独立运行；OASIS仿真调用流程存在但未与叙事阶段判断打通；新闻数据严重不足（最多10条/股）。
主要风险: ①新闻数据是瓶颈（AkShare严格限速）；②叙事主题识别需要跨股票聚合，当前逐股处理丢失全局视角；③生命周期打分无法回测（历史叙事无标注）。

3.2 为什么 S01/S03/S10/S11 无法充分验证
S01（供应链传导）— 根因：图谱覆盖极度不足

图谱96条边，涉及不超过~120家公司，CSI800共800家
能被图谱遍历覆盖的"上游-下游对"极少
近期公告回测3天（3/15-3/17）：1491条公告，0个供应链事件被确认
即使策略逻辑100%正确，图谱缺失就等于永远0信号
S03（事件传播）— 根因：与S01共享图谱缺陷 + 逻辑重叠

依赖同一图谱，同样稀疏
事件传播模型与S01高度重叠，双倍调用LLM但无增量信息
需要先解决图谱问题，再重新设计
S10（情绪模拟）— 根因：新闻数据瓶颈

ak.stock_news_em 每只股票最多10条新闻
800只股票 × 10条 = 8000条，OASIS仿真可运行，但信号质量受限于信息密度
市场热点API（stock_hot_rank_em等）网络限制，无法获取全局情绪背景
OASIS仿真在缺乏足够信息的股票上产生"噪声信号"
S11（叙事追踪）— 根因：跨股票聚合缺失 + 历史标注缺失

当前逐股处理，无法识别"跨股票的主题叙事"（如"低空经济"涉及100+股票）
叙事生命周期判断需要历史时序数据（至少3-6个月积累）
无历史叙事标注数据，无法做回测验证

Phase 1–3 实际审计结果
A. 执行摘要
最重要发现（非方法论，是真实代码审计结论）：

S07"图因子"策略的A级评分（Sharpe 1.87）与图谱完全无关。
所有6个图谱因子均为零值。S07实际运行的是纯传统量化多因子（动量/反转/波动率/换手率/PE/ROE）。这不是小问题——这意味着项目迄今为止唯一验证有效的策略，其有效性与MiroFish的核心差异化能力（图谱）毫无关联。

其他关键事实：

11个策略中，只有2个进入了全量CSI800回测（S01、S07），其余9个均未经严格验证
6个已有信号输出的策略（S05/S08/S09/S10/S11/S02），其信号质量存在严重问题（见Phase 1）
AkShare的新闻API（stock_news_em）、公告API（stock_notice_report）、市场热点API均在今日测试中失败
唯一稳定可用的行情数据：stock_zh_a_hist（日OHLCV）
本地数据持久化：除图谱JSON和信号JSON外，无任何历史因子数据库、无CSV、无Parquet

关键发现：S07图谱因子全部为零
通过代码检查确认：


供应链中心性 (supply_chain_centrality) → 0
  原因: _build_code_name_maps() 查找 attrs["stock_code"]
        但图谱节点存的是 attrs["code"]
        → name_to_code 字典为空 → PageRank找不到任何股票

机构集中度 (institution_concentration) → 0
